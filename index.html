1. Predictive Task

The objective of this project is to predict the final score (upvotes minus downvotes) of a Reddit post at the moment it is submitted. This is framed as a supervised regression problem.

Input Features

  • Post title (raw text)
  • Subreddit (categorical community identifier)
  • Title length: number of characters
  • Word count: number of words in the title
  • Posting time: hour of day, day of week, and month

Target Variable

The target is the Reddit score of each post, defined as:

score = upvotes − downvotes real-valued output heavy-tailed distribution

2. Data & Exploratory Analysis

Score Distribution

The score distribution is highly skewed with a long right tail: most posts receive relatively modest scores, but a small fraction of posts go "viral" and receive very high scores.

Subreddit Popularity

Different subreddits exhibit very different typical scores and activity levels. Some communities have higher mean scores but fewer posts, while others have more moderate scores but large volumes of content.

3. Feature Engineering & Preprocessing

I construct a feature representation that captures content, community context, and posting time. These features are combined in a single scikit-learn Pipeline.

Final Feature Set

text_features = ["title"] → TF–IDF (max_features = 5000, English stopwords) categorical_feats = ["subreddit"] → OneHotEncoder(handle_unknown = "ignore") numeric_features = ["title_len", "word_count", "hour", "dayofweek", "month"]

Numeric features use mean imputation for any missing values. Text and categorical transformations are handled by a ColumnTransformer, which keeps preprocessing consistent between training and evaluation without leaking test data.

4. Models

I evaluate simple baselines and two linear regression models:

Baselines

  • Global mean baseline – always predicts the mean training score.
  • Global median baseline – always predicts the median training score.
  • Subreddit mean baseline – predicts the mean score for each subreddit.

Machine Learning Models

  • Linear Regression – least squares regression in the combined feature space.
  • Ridge Regression – linear model with L2 regularization to reduce overfitting.

Regularization is particularly important because TF–IDF creates a high-dimensional feature space with many correlated predictors. Ridge controls coefficient magnitude and improves generalization.

5. Results & Error Analysis

Model Comparison (RMSE)

All regression models are evaluated on a held-out test set using mean absolute error (MAE) and root mean squared error (RMSE).

Actual vs Predicted (Ridge)

The Ridge model’s predictions generally align with the diagonal, especially for typical scores. The largest discrepancies occur for extremely high-scoring posts, which are rare and hard to predict.

Residuals (Ridge)

Residual plots show that errors are centered around zero but have a heavy tail, reflecting the underlying score distribution. The model is well-calibrated on average but struggles with outlier posts.

6. Discussion & Related Work

Predicting social media engagement has been studied on platforms such as Reddit, Twitter, and news sites. Common approaches use text features (bag-of-words or embeddings), temporal patterns, and community metadata. My approach fits into this line of work by combining TF–IDF title features with subreddit and time-of-post signals.

Key Takeaways

  • Ridge regression outperforms all baseline models and plain linear regression.
  • Text and subreddit identity carry substantial predictive power.
  • The heavy-tailed score distribution makes viral posts difficult to predict precisely.

Future Directions

  • Experiment with transformer-based text embeddings or pretrained language models.
  • Incorporate author history or comment dynamics as additional context.
  • Use robust loss functions or quantile regression for heavy-tailed targets.
